# Task Completion Protocol

When a task is completed:

1.  **Verify:** Ensure the code compiles/runs and meets the requirements.
2.  **Lint/Format:** Run necessary linting and formatting commands.
3.  **Test:** Run relevant tests to ensure no regressions.
4.  **Update Documentation:** Update Plan files (mark as done), `lastchanges.md` (if used), and any relevant docs.
5.  **Commit:** Stage and commit changes with a descriptive message.
6.  **Report:** Inform the user of the completion and any next steps.
